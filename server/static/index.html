02 Environment Architecture
01 / Environment
Partial Observability
Patient data is hidden. The agent must request fields one at a time, receiving answers from the environment server. This creates a genuine exploration-exploitation tradeoff.
FastAPI WebSocket HF Spaces Docker
02 / Agent
Chain-of-Thought Reasoning
The LLM agent reasons about which field to ask next based on revealed data, targeting the most likely disqualifier first to minimize question cost.
Qwen2.5-72B OpenAI SDK HF Router
03 / Reward
Efficiency-Incentivized Grading
Every question costs −1 reward. Correct decisions earn +20, wrong decisions −20. Bonus +2 for citing specific values in the reason, incentivizing interpretable decisions.
Sparse Reward Partial Credit Interpretability
03 Reward Structure
Event Reward Rationale
Each question asked −1 per question Incentivizes efficient information gathering
Correct eligibility decision +20 Primary objective signal
Wrong eligibility decision −20 Symmetric penalty for incorrect decisions
Reason cites specific value +2 Rewards interpretable, grounded reasoning
Hitting max steps (10) without deciding −5 Penalizes indecision
Partial criteria met (medium task) +1 per criterion Encourages partial progress
